# 프로젝트 개요

## 프로젝트 소개

쿠폰 서비스는 이커머스 플랫폼을 위한 독립적인 쿠폰 관리 마이크로서비스입니다. 쿠폰 생성, 발급, 사용, 통계 등 쿠폰 라이프사이클 전반을 관리합니다.

## 프로젝트 목적

### 비즈니스 목적
- 마케팅 프로모션을 위한 유연한 쿠폰 시스템 제공
- 고객 획득 및 유지를 위한 타겟팅 쿠폰 발급
- 실시간 쿠폰 사용 현황 모니터링

### 기술적 목적
- 대용량 트래픽 처리 가능한 확장성 있는 시스템
- 동시성 제어를 통한 쿠폰 중복 발급 방지
- 이벤트 기반 아키텍처를 통한 느슨한 결합

## 핵심 기능

### 쿠폰 정책 관리
- 다양한 할인 타입 지원 (정액, 정률)
- 발급 조건 설정 (기간, 수량, 대상)
- 쿠폰 코드 자동 생성

### 쿠폰 발급
- CODE 타입: 사용자가 쿠폰 코드 입력
- DOWNLOAD 타입: 사용자가 직접 다운로드
- FIRST_COME 타입: 선착순 발급 (Redis 분산 락 활용)
- EVENT 타입: 이벤트 기반 자동 발급
- 중복 발급 방지 메커니즘
- 멱등성 보장

### 쿠폰 사용
- 결제 시스템과의 Kafka 이벤트 연동
- 예약 기반 2단계 커밋 (예약 → 사용 확정)
- 자동 타임아웃 처리 (10분)
- 롤백 메커니즘
- 멱등성 보장으로 중복 처리 방지

### 통계 및 모니터링
- 실시간 발급/사용 통계
- 쿠폰별 성과 분석
- 사용자별 쿠폰 현황

- **API Style**: RESTful API

### Infrastructure
- **Container**: Docker
- **Orchestration**: Docker Compose
- **CI/CD**: GitHub Actions

## 시스템 요구사항

### 성능 요구사항
- 초당 5,000건 이상의 쿠폰 발급 처리
- API 응답 시간 P95 < 100ms
- 99.9% 가용성
- Kafka 처리량: 10,000 msg/s
- 동시 사용자: 10,000+

### 확장성 요구사항
- 수평적 확장 가능한 구조
- 무중단 배포 지원
- 멀티 인스턴스 동시 운영

### 보안 요구사항
- API Gateway를 통한 인증 (X-User-Id 헤더)
- 쿠폰 코드 암호화
- SQL Injection 방지
- 감사 로그 기록
- Rate Limiting (분당 100회)

## 최근 개선사항 (2024.12)

### 성능 최적화
- **Kafka 동시 처리**: 3 → 10 스레드로 증가
- **배치 처리**: 100건 단위 일괄 처리 도입
- **처리 시간**: 평균 200ms → 60ms로 80% 개선

### 안정성 개선
- **ACK 전략 개선**: 비즈니스 로직 실패 시 재처리 가능
- **멱등성 보장**: 중복 이벤트 처리 방지
- **타임아웃 로직**: reservationId 보존으로 늦은 이벤트 처리

### 코드 품질
- **검증 강화**: 구체적인 오류 메시지 제공
- **테스트 커버리지**: 85% 이상 유지
- **PostgreSQL 마이그레이션**: 실제 운영 환경과 동일한 DB 사용
